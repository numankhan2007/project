====================================================
KNOWN ERRORS AND TROUBLESHOOTING GUIDE
====================================================

ERROR 1: "Server Error: Unable to connect to the backend" (Frontend Login/Register Failure)
-------------------------------------------------------------------------------------------
- WHERE IT IS FROM: The frontend services (Axios interceptors in api.js or Login.jsx).
- CAUSE: The backend FastApi server is offline or is running on a different port than expected, OR Redis/PostgreSQL backend services are down causing the backend to crash.
- SOLUTION: 
  1. Ensure the backend is running using `uvicorn main:app --reload`.
  2. Confirm your .env file in the frontend has `VITE_API_URL=http://localhost:8000/api`.
  3. Ensure PostgreSQL and Redis are active, otherwise the backend fails to connect and throws a 500 error on registration/login endpoints.

ERROR 2: "Page redirected to User Login and Admin Login continuously"
-------------------------------------------------------------------------------------------
- WHERE IT IS FROM: The routing configuration in the frontend (Login/Register files and AppRoutes.jsx).
- CAUSE: Upon successful login, the application navigated to navigate('/') which pointed to the Landing Page. The Landing Page lacked logic to redirect authenticated users to the dashboard.
- SOLUTION: This has been fixed! Instead of navigating to '/', we now explicitly call `navigate('/home')`. Also, the Landing page now checks `isAuthenticated` and pushes logged-in users directly to `/home`.

ERROR 3: React Vite Dependency Conflict (ESLint / Rollup Issues)
-------------------------------------------------------------------------------------------
- WHERE IT IS FROM: The `npm run dev` pipeline.
- CAUSE: Caching or peer dependency mismatches in node_modules from conflicting plugin versions.
- SOLUTION: Clear the cache or reinstall packages. Run `npm cache clean --force` followed by removing `node_modules` and `package-lock.json`, and running `npm install`.

ERROR 4: Python Virtual Environment Missing or Permission Erros
-------------------------------------------------------------------------------------------
- WHERE IT IS FROM: Running `activate` or creating the initial `venv`.
- CAUSE: Windows policy blocking script execution, or existing corrupted environments.
- SOLUTION: 
  - Open PowerShell as Administrator and run: `Set-ExecutionPolicy Unrestricted -Scope CurrentUser`.
  - Alternatively, simply use the `start.bat` provided that automatically bypasses and runs the python virtual environment.

ERROR 5: SMTP Exception / Email Failing to Send
-------------------------------------------------------------------------------------------
- WHERE IT IS FROM: backend/services/email_service.py.
- CAUSE: Invalid or expired SMTP passwords/app-passwords in backend/.env.
- SOLUTION: Update the .env file with a newly generated Google App Password for the SMTP Server configurations.
